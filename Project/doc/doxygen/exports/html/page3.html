Makefile description:

Makefile has the main purpose of automating software compiling/linking operations.

Makefile details:

In this project, makefile purposes has been extended in order to take care of further actions to simplify testing operations and documentation management. So the makefile can handle different tasks like: compiling/launching the software, to make/view doxygen documentation and displaying LaTeX abstract pdf. As can be seen inside makefile (accessible in 'makefile' section of doxygen 'File list'), the file has been divided in chapters and sub-chapters to be better explained in this page:

  • (1) Variables and files:
    • (1.1) Variables to compile and run software: compiling flags for libraries and main program, for linking and command to run the software.
    • (1.2) Source files: graph-library, user-interface-library, files-library, timer-library and main program code files.
    • (1.3) Header files: graph-library, user-interface-library, files-library and timer-library header files.
    • (1.4) Output files: software executable, graph-library, user-interface-library, files-library, timer-library and main program object files.
    • (1.5) Doxygen documentation: doxygen-configuration file, doxygen-exports folder and doxygen main html page.
    • (1.6) Abstract pdf file (from LaTeX): LaTeX abstract exported (.pdf) file, containing theoretical aspects about Dijkstra's algorithm.

  • (2) Make commands
    • (2.1) Compile and link all files: command to compile and link all software's files.
    • (2.2) Run test software (main program): command to run test software.
    • (2.3) Compile-link all files and run test software (main program): chapter (2.1) and chapter (2.2) in one shortcut-command.
    • (2.4) Commands to compile and link software step-by-step: commands to compile graph-library, user-interface-library, files-library, timer-library, main program and command to link all the software's files.
    • (2.5) Clear: command to clear all the software's generated files.
    • (2.6) Export/view/clear doxygen html documentation: command to view documentation in firefox, command to re-generate documentation, shortcut-command to re-generate and view documentation in firefox, command to clear documentation, and shortcut-command to clear and re-generate documentation.
    • (2.7) Open abstract pdf file (made with LaTeX): command to open 'relazione-finale-CRISTIAN-MERLI.pdf' file using the predefined pdf-viewer software.

Available makefile commands:

Note: in order to correctly execute makefile's commands, the user must be inside 'Project/sw/' folder (same folder of the makefile).
Note: makefile commands-syntax make [command], available commands listed below.

  • Software compiling commands:
    • make: command to compile and link the whole project to executable (shortcut command of the following one).
    • make all: same command as above, full-name command.
    • make run: command to launch test-software executable.
    • make all_run: shortcut-command to compile/link the whole project to test-software executable and to run it.
    • make clr: command to clear all the software's generated files.
  • Software step-by-step compiling commands:
    • make compile_lib_graph: command to compile only the graph-library.
    • make compile_lib_ui: command to compile only the user-interface-library.
    • make compile_lib_files: command to compile only the files-library.
    • make compile_lib_timer: command to compile only the timer-library.
    • make compile_main: command to compile only the main program (test software).
    • make link_all: command to link all libraries and main program to test-software executable.
  • Doxygen documentation commands:
    • make docs: command to open firefox in order to display doxygen html documentation.
    • make docs_gen: command to re-generate doxygen html (and rtf) documentation.
    • make docs_gen_view: shortcut-command to re-generate doxygen html (and rtf) documentation and to open firefox displaying doxygen html documentation.
    • make docs_clr: command to clear generated doxygen html (and rtf) documentation.
    • make docs_clr_gen: shortcut-command to clear and re-generate doxygen html (and rtf) documentation.
  • LaTeX abstract (pdf) command:
    • make abstract: command to open and view LaTeX exported pdf file through the predefined pdf-viewer software.
